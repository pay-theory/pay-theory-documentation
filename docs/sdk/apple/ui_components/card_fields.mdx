# Pay Theory Card UI Fields

This documentation covers all the SwiftUI views provided by the Pay Theory SDK for capturing card information. These views are designed to be used within a `PTForm` and require a `PayTheory` object to be provided as an environment object.

## Common Requirements

All of these views:
- Must be used within a `PTForm` view hierarchy
- Require a `PayTheory` object to be provided as an environment object
- Automatically handle data binding and validation with the Pay Theory SDK

## PTCardName

A SwiftUI view that provides a text field for capturing the cardholder's name.

### Usage

```swift
PTCardName(placeholder: "Name on Card")
```

### Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| placeholder | String | "Name on Card" | The placeholder text displayed when the field is empty |

## Example Usage

Here's an example of how to use these fields together within a `PTForm`:

```swift
struct CardPaymentView: View {
    @EnvironmentObject var payTheory: PayTheory

    var body: some View {
        PTForm {
            VStack(spacing: 20) {
                PTCardName()
                PTCardNumber()
                HStack {
                    PTExp()
                    PTCvv()
                }
                PTCardLineOne()
                PTCardLineTwo()
                PTCardCity()
                HStack {
                    PTCardRegion()
                    PTCardPostalCode()
                }
                PTCardCountry()
            }
        }
        .environmentObject(payTheory)
    }
}
```

This example creates a complete card payment form using all the available card UI fields. Remember to provide the `PayTheory` object as an environment object to the `PTForm` for proper functionality.
